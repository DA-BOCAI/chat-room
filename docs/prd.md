# 需求文档

## 1. 应用概述

- **应用名称**：多人聊天室
- **应用描述**：一款需要用户注册登录后方可访问的多人聊天室系统，支持公开聊天室与私密聊天室两种房间类型，私密聊天室需输入房间密码方可进入，支持房间创建与管理、实时消息收发、在线用户列表与人数实时展示。不同账号的用户可通过链接访问并登录，进入同一聊天室进行实时交流。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 需要在多人环境下进行实时文字沟通的用户群体

### 2.2 核心使用场景
- 用户通过链接访问系统，注册账号后通过账号密码登录
- 登录后可在大厅查看公开房间列表，选择加入公开聊天室或输入密码进入私密聊天室
- 不同账号的用户可同时进入同一聊天室，进行实时文字交流
- 用户可创建公开或私密聊天室，房间创建者可解散房间，普通用户可退出房间

---

## 3. 页面结构与功能说明

### 3.1 整体页面结构

```
多人聊天室
├── 注册页
├── 登录页
├── 聊天室大厅页（登录后可访问）
│   ├── 房间列表（公开 + 私密）
│   └── 创建房间入口
└── 聊天室详情页（进入房间后）
    ├── 消息区域
    ├── 消息输入区
    └── 在线用户列表
```

### 3.2 注册页

**功能说明：**
- 提供用户名输入框（唯一性校验）
- 提供密码输入框（不少于6位）
- 提供确认密码输入框（与密码一致性校验）
- 注册成功后自动跳转至登录页
- 已有账号可点击跳转至登录页

**字段说明：**

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|----------|------|
| 用户名 | 文本输入 | 是 | 唯一，不可重复 |
| 密码 | 密码输入 | 是 | 不少于6位，加密存储 |
| 确认密码 | 密码输入 | 是 | 须与密码一致 |

### 3.3 登录页

**功能说明：**
- 提供用户名与密码输入框
- 登录成功后跳转至聊天室大厅页
- 未注册用户可点击跳转至注册页

### 3.4 聊天室大厅页

**功能说明：**
- 仅登录用户可访问，未登录用户访问时自动跳转至登录页
- 展示当前所有房间列表（公开与私密），每个房间显示：房间名称、房间类型（公开/私密）、当前在线人数
- 点击公开房间直接进入；点击私密房间弹出密码输入框，验证通过后进入
- 提供「创建房间」入口，填写房间名称并选择房间类型（公开/私密），选择私密时需设置房间密码
- 顶部展示当前登录用户名及「退出登录」按钮

### 3.5 聊天室详情页

**功能说明：**
- 仅登录用户可访问，未登录用户访问时自动跳转至登录页
- 同一聊天室支持多个不同账号的用户同时在线并实时交流
- **消息区域**：实时展示房间内所有用户发送的消息，每条消息显示发送者用户名、发送时间及消息内容
- **消息输入区**：文字输入框 + 发送按钮，支持回车键发送
- **在线用户列表**：实时显示当前房间内在线用户列表及在线人数
- **退出房间**：普通用户可退出当前聊天室，返回大厅页
- **解散房间**：房间创建者可解散房间，解散后房间内所有用户自动返回大厅页，房间从列表中移除

## 4. 业务规则与逻辑

### 4.1 登录状态管理
- 用户登录后生成会话令牌，存储于客户端
- 会话超时后，用户下次操作时自动跳转至登录页，并提示「登录已过期，请重新登录」
- 用户主动点击「退出登录」后，立即清除会话令牌，跳转至登录页
- 每个用户须以独立账号登录，不支持匿名访问

### 4.2 多用户同室规则
- 同一聊天室支持多个不同账号的用户同时在线
- 每个用户以其注册的用户名作为唯一身份标识，消息发送时展示对应用户名
- 每个用户同一时间只能处于一个聊天室中

### 4.3 房间类型与密码规则
- 房间分为公开与私密两种类型，创建时由创建者指定
- 私密房间需设置房间密码（不少于4位），其他用户进入时须输入正确密码方可加入
- 房间密码由创建者设定，创建后不可修改
- 大厅列表中私密房间以明显标识区分（如加锁图标或「私密」标签）

### 4.4 房间管理规则
- 房间创建者拥有解散房间的权限，普通用户无此权限
- 房间解散后，房间内历史消息不再保留
- 用户退出或加入房间后，在线用户列表与在线人数实时更新

### 4.5 消息规则
- 消息内容不得为空，发送前进行非空校验
- 消息长度限制为500字以内
- 消息按发送时间升序排列，新消息自动滚动至底部

### 4.6 安全规则
- 密码在存储前进行加密处理，不可逆
- 所有用户输入内容进行转义处理，防止跨站脚本攻击
- 聊天室内容仅对当前房间内的已登录用户可见
- 未登录状态下无法访问大厅页及聊天室详情页

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户名已被注册 | 注册页提示「该用户名已存在，请更换」 |
| 密码与确认密码不一致 | 提示「两次输入的密码不一致」 |
| 用户名或密码错误 | 登录页提示「用户名或密码错误」 |
| 进入私密房间时密码错误 | 提示「房间密码错误，请重新输入」 |
| 消息内容为空时点击发送 | 提示「消息内容不能为空」，不发送 |
| 消息超过500字 | 输入框限制最大字符数，超出部分无法继续输入 |
| 网络断开时发送消息 | 提示「网络连接异常，消息发送失败，请检查网络后重试」 |
| 房间被创建者解散时 | 房间内所有用户收到提示「该聊天室已被解散」，自动返回大厅页 |
| 未登录用户直接访问大厅或聊天室页面 | 自动跳转至登录页，并提示「请先登录」 |
| 会话超时后操作 | 自动跳转至登录页，提示「登录已过期，请重新登录」 |
| 未登录用户通过链接直接访问聊天室 | 自动跳转至登录页，登录成功后可继续访问目标页面 |

---

## 6. 验收标准

- 用户可成功完成注册，用户名唯一性校验生效，密码加密存储
- 用户可通过账号密码成功登录，登录失败时提示准确
- 未登录用户访问受保护页面时自动跳转至登录页
- 登录用户可在大厅查看房间列表，房间类型（公开/私密）标识清晰
- 用户可成功创建公开房间，直接进入；创建私密房间时可设置密码
- 进入私密房间时密码验证生效，密码错误时提示并阻止进入
- 不同账号的用户可同时进入同一聊天室，各自以注册用户名发送消息，消息实时互通
- 用户可进入聊天室，实时收发文字消息，消息展示发送者用户名与时间
- 在线用户列表与在线人数实时更新，数据准确
- 房间创建者可解散房间，房间内用户自动返回大厅
- 普通用户可退出房间，返回大厅
- 会话超时机制正常生效，超时后跳转登录页
- 所有用户输入内容经过转义处理，防止脚本注入
- 通过链接访问的未登录用户被引导至登录/注册页，完成登录后可正常使用
